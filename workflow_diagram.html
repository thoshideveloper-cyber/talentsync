Anti-Hallucination Rules — R1 to R11 (enforced in code, not in prompts)
R1 Grounding quote always visible in UI for every seniority call (Board detail panel)
R2 Quote gate: ≥25 chars + _norm(quote) in _norm(text). Fail → Uncertain override
R3 Word-boundary skill filter: (?<![a-z0-9])term(?![a-z0-9]) + synonym canonical map
R4 UNCERTAIN returned when signals absent — no "infer closest tier" fallback in prompt
R5 Deterministic enrichment (bias, pay, mismatch): 0 LLM calls, purely rule-based
R6 temperature=0 for all LLM calls — reproducible, consistent outputs
R7 JSON-mode response (response_mime_type=application/json); Pydantic v2 schema enforcement
R8 Retry ≤2 per slot. Rotate key/model on quota; don't retry wrong answers
R9 Hallucination rate reported pre-filter AND post-filter in eval.py (honest accounting)
R10 Grounding receipt always rendered in Board detail — user can see the exact source text
R11 one_line_summary labeled "indicative, not verified" everywhere it appears — dashed amber border in UI, footer disclaimer, CSV note. Never presented as ground truth.
